# UX- Engineer 定义以及 JD

# 定义

A user experience (UX) engineer builds and designs digital products like websites and apps while using UX principles. UX engineers generally deal with the front end of these products—the user-facing parts that people interact with, like the buttons, layouts, and flow of the interfaces.(用户体验(UX)工程师在使用UX原则的同时构建和设计像网站和应用程序这样的数字产品。UX工程师通常处理这些产品的前端——人们与之交互的面向用户的部分，如按钮、布局和界面流。)

# 前端转行 UXE 需要注意的问题

# （1）了解规范

现如今转行做 UX 设计师的人越来越多，其中很多设计师非科班出身。新人设计师通常以一种抽象的方式通过临摹 Dribble 或 Behance 的 UI 作品来自学，但如果交互的基础不扎实，可能会适得其反。

在针对特定平台（例如 iOS 或 Android）进行设计时，了解设计的基础知识尤为重要。当我对使用某种交互不确定时，我经常会去参考一些主流的设计系统，例如 Apple 的人机界面指南或 Google 的 Material Design。随着对设计系统的逐渐熟悉，你会慢慢了解到在你的设计里什么是必要的，以及使用哪些组件。

# （2）保持灵感

在创意领域，最好能通过多看其他设计师的作品来保持灵感。我的方法是去找我认为优秀的 App，用的网站是 Mobbin 和 Product Hunt。当我选定了一个 App 之后，我会在网上试着搜索它的设计者，并看他们是否谈到其设计过程。无论是通过作品集还是博客文章，了解他们如何制定设计决策都对你很有帮助，通过了解他们的思考过程，能帮助你在解决自己的设计问题时发散思维。

在我灵感枯竭时，这帮助我从自己的工作中退一步思考，并挖掘设计可以创造出的无限可能。
