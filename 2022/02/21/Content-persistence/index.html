Content persistence
从广义上讲，今天的 Web 浏览是通过所谓的位置寻址来操作的（就是通过 url 跳转）。位置寻址从 Web 上的特定位置检索在线信息，即从 URL 跳转检索。但是，这有其明显的缺点。位置寻址是中心化的，这意味着控制该位置的任何人都控制内容。位置寻址 URL 跳转后的任何内容都可以更改：文章的语言，数字图稿中使用的颜色或记录身份的属性。简而言之，位置寻址的网址是可被人为利用的.

此问题的解决方案从内容寻址（content addressing）开始。基于内容的寻址允许您基于唯一的一串 hash 值；无论它存储在何处，如果您拥有那串 hash，则应该能够检索内容。这意味着，如果您使用内容寻址存储 — 特别是 IPFS， 您的内容不再从网络上的单个位置检索（location addressing）。相反，当有人请求该内容时，将从 IPFS 网络上具有该内容副本的任何参与节点中检索该内容。内容片段可以位于许多节点上，内容可以完全从一个节点检索，也可以从多个节点以零碎的形式组合。

但是，内容寻址只是解决方案的一部分。仅仅因为数据无论它存储在何处都可以通过 IPFS 检索，这并不意味着数据可以保证永远存在。要获得完整的解决方案，内容持久性是关键。

# 为什么选择内容持久性

我们前面论证了内容寻址（content- addressing）代替位置寻址（location-addressing）能解决定位的内容被替换或者说链接失效的问题，那现在问题又来了：我们如何确保内容在任何时候都可以继续被用？换句话说，我们如何确保内容持久存在？如果没有能让内容存储具备持久性，那么是内容寻址（content- addressing）的网络也会遭受与今天类似的危险；除非内容持续存在，否则我们将面临碎片化，不完整和被遗忘（失效）的网络的风险。

与数据和信息领域的大多数解决方案一样，有中心化和去中心化的两个方案可用于解决内容持久性问题。中心化的方案是运行一个服务器去持续把内容存储到其之上。但是，中心化存储无法实现真正的持久性，因为它会受到单个中心点故障的影响。

另一种方案，是去中心化的内容持久性方案，是确保内容随着时间的推移一直能保持可用的唯一方法。使用完全独立的、可互操作的节点来存储由强加密保证支持的数据，可以防止信息因任何中心化操作或者不作为而变得不可用的问题。如何做到这一点？Filecoin 通过建立内容持久化的基础和在此之上创造一系列工具和服务，从而在这个行业做到领先。

# 用于内容持久性的文件币

将 Filecoin 与 IPFS 结合使用，通过将内容持久性的激励层与 IPFS 的内容寻址解决方案相结合，提供了一个完整的解决方案。IPFS 确保内容在没有明确的审计跟踪的情况下不会随时间而变化，从而解决了 URL 无法解析的问题。Filecoin 通过确保内容持久性以保证可供检索，确保 IPFS 提供的基于内容的寻址随着时间的推移保持弹性（可复原力）。

Filecoin 通过以下方式实现真正的使数据分散存储存储在 Filecoin 网络上：新型密码学、共识协议和博弈论激励 （ novel cryptography, consensus protocols, and game-theoretic incentives）。这一切的核心是 Filecoin 独特的存储验证方法。

Filecoin 的存储验证系统解决了去中心化存储以前难以解决的问题：存储提供商如何证明他们确实在存储他们所说的数据，并且他们正在为此提供独特的物理空间？

对于我们前面提到的那些中心化存储提供商–用户信任那些保证其系统完整性和安全性的知名公司。在 Filecoin 网络上，世界上任何人都可以提供存储空间。为了在像 Filecoin 这样的去中心化网络上保持信任，您需要一种方法来建立对网络本身的信任。为此，任何想要在 Filecoin 的分散式网络上提供经过验证的存储的人都需要证明两件事 - 首先，正确的数据集存储在给定的存储空间中，其次，同一组数据在给定的时间内是持续性地在被存储的。

文件币的证明算法 负责此验证。 Proof-of-Replication 算法可以证明一个供应商正在把一个用户的原始数据的唯一副本以物理形式存储起来，而 Proof-of-Spacetime 则证明客户端的数据在一段连续时间内被存储着。

除了这个证明系统之外，Filecoin 网络还依赖于博弈论激励来阻止恶意或疏忽活动。所有 Filecoin 存储提供商在同意成为提供商时必须以 Filecoin 令牌（FIL）的形式提供抵押品。任何未能通过 Proof-of-Spacetim 验证检查的存储提供商都会受到惩罚，失去一部分抵押品，并最终被阻止再次向客户提供存储。

Filecoin 网络继续增长，为内容持久性、内容寻址存储提供了快速增长的容量。您可以查看最新的高级存储指标，以及其他 Filecoin 数据和区块链浏览器，点击这里可以查看数据 filfox.

# 那 nfts 在此基础上受益

NFT 是当今加密领域增长最快的领域之一 - 然而，当谈及到持久性和可靠性时候，nfts 就是最被关注的审查对象。

当我们谈论 "铸造 NFT" 或 "交易 NFT" 时，我们通常指的不是创意作品本身，而是作品的记录 - 换句话说，不是内容（构成作品的颜色，形状或声音），而是元数据（例如描述性文本，艺术家数据信息或指向内容本身位置的指针）。但是，该内容和数据都不会自动存在于区块链上。如果许多 NFT 的内容和元数据未可靠地存储，这将使其面临解决可用性和持久性的问题。使用 IPFS 解决 NFT 的解决问题正迅速成为常态，并且一个庞大且不断增长的生态系统固定服务帮助确保 IPFS 数据的可用性。

具体到 NFT 领域，nft.storage 是一项免费服务，旨在使处理内容寻址和持久性尽可能简单。该服务允许任何人生成元数据，只需几行代码即可免费在 Filecoin 上铸造和存储 NFT。开发人员注册一个帐户，生成 API 访问密钥，并可以使用简单的客户端库来生成元数据并永久存储其 NFT。

使用上面概述的 Filecoin + IPFS 方法，通过 nft.storage 存储的 NFT 不仅可以通过 IPFS 网络获得，而且还可以通过 Filecoin 激励其全球存储提供商网络继续长期存储 NFT 内容和元数据来保护它们不会消失。

# 未来发展

目前，保持内容持久存储对于个体与那些保证内容在线存储的节点之间而言是一种经济上的关系。事实证明，该模型对于 Filecoin 生态系统具有极大的可持续性。然而，生态系统也在为更多的人做准备，他们希望将他们的信息存储在 Filecoin 上，并创建解决方案来适应去中心世界的所有权问题和支付方式。nft.storage 服务就是一个例子：通过为 NFT 应用程序的开发人员提供存储，nft.storage 还鼓励 NFT 最佳实践，使它们尽可能易于实现，并消除了创建者和收集者的经济考虑。

随着时间的推移，更多的生态系统参与者将出现，以激励重要数据的持久性，在 IPFS / Filecoin 生态系统中扮演不同的经济角色，并使个人和存储提供商之间的基本关系更加多样化，灵活和可持续。即将出现的一个例子是 DataDAOs，一种去中心化的的自治组织，专门用于存储、维护、处理和许可潜在的大规模数据集。DataDAO 十分胜任塑造分布式系统上数据存储和支付的发展的这一个工作；通过承担存储大型数据集的技术和财务责任，他们可以有效地将关键数据货币化，同时保持对去中心化内容持久性的承诺。其他，像 Ocean Protocol and Filehive，不作为 DAO 运营，但扮演着与生态系统参与者类似的角色，这些参与者依靠 Filecoin 来维护他们正在货币化的数据并授权给他人。
